论文一：分步执行计划（含验证门槛）
目标: 完成一篇 Cell Reports 级多组学发现论文，7张主图，≤7000字
核心论点: 急性心肌I/R中，Ogt下调通过RELA/NF-kB间接激活Nrf2，髓系SPP1-CD44旁分泌放大铁死亡信号，铁蛋白自噬加剧心肌损伤。该机制是急性损伤阶段特异性的。
阶段一：数据与代码就绪 (Day 1-2)
1.1 分析脚本归档
输入: scripts/*.py (step1-4 分析脚本)
操作: 
清理分析脚本中的临时路径和调试代码
统一文件路径为相对路径 (基于项目根目录)
添加 argparse 支持，可从命令行调用
每个脚本添加 docstring 说明输入/输出/依赖
输出: scripts/ 下每个 .py 文件可独立运行，python stepX_xxx.py 即可复现
验证: 在新终端中逐个运行每个 step 脚本，确认无报错且输出文件与之前一致
1.2 创建 GitHub 仓库
输入: 清理后的脚本 + 输出文件
操作: 
创建 README.md (研究概述 + 环境配置 + 运行说明)
创建 environment.yml 或 requirements.txt
上传 scripts/ + output_tables/ (不含大数据文件)
STAR Methods 中引用 GitHub URL
输出: github.com/<user>/Ogt_Nrf2_MultiOmics (公开或审稿期间私密)
验证: 另一台机器上 git clone 后能按 README 复现核心分析
1.3 数据可访问性确认
输入: 所有GEO/PRIDE accession numbers
操作: 
确认 GSE193997, GSE240848, GSE163129, GSE307385 在 GEO 中为公开状态
确认 PXD035154, PXD040135, PXD053525 在 ProteomeXchange 中为公开状态
确认 GSE57338 (外部验证) 为公开状态
在 Data Availability 段落中列出所有 accession
输出: 完整的 Data Availability 段落，每个数据集都有可访问的 accession number
验证: 用浏览器逐一打开每个 accession URL，确认可公开访问

阶段三：手稿撰写 (Day 8-21)
3.1 STAR Methods (Day 8-9)
目标: 另一实验室能根据Methods完全复现你的分析
Key Resources Table (抗体/试剂/软件/数据 accession)
Experimental Model: 小鼠I/R模型参数，样本数，时间点
Method Details: RNA-seq预处理(STAR, featureCounts), DESeq2参数, snRNA-seq(Cell Ranger, Seurat), 蛋白组(MaxQuant), CellChat参数, NicheNet参数, WGCNA参数
Quantification: 统计检验方法，P值校正方法，GSEA参数，Fisher精确检验
Data and Code Availability: GEO/PRIDE accession, GitHub URL
验证: 让不了解该项目的人阅读Methods后列出所需软件和数据，与实际一致
3.2 Results 第一轮 — 叙事草稿 (Day 10-13)
目标: 7个子标题，每段500-800字，全部按'问题->数据->结论'结构
只写文字，不嵌入Figure引用细节（先保证叙事流畅）
每段第一句必须是一个科学问题/假设
每段最后一句是结论+过渡到下一段
写完一段后立即大声朗读——不顺口就改
验证: 连续朗读全部Results，不卡顿、不跳跃、逻辑自洽。卡住的地方重写。
3.3 Results 第二轮 — 数据锚定 (Day 14-15)
目标: 每个claim都有Figure/Table支撑，无空口无凭的陈述
逐句标注Figure引用
检查：每个主Figure在Results中至少被引用2次
补充Statistics: 每个差异表达/富集都报告 effect size + P value
检查：所有数字在文中的报告与Figure中一致
验证: 打印Results，用荧光笔标出每个Figure引用；未被引用的Figure需删除或补充文字
3.4 Introduction (Day 16)
目标: 3段，400字，漏斗结构
Para 1: 心肌I/R的临床重要性 + O-GlcNAc修饰的研究现状
Para 2: Nrf2/铁死亡在I/R中的已知角色 + 知识缺口(Ogt如何连接Nrf2? 细胞类型特异性? 翻译后调控?)
Para 3: 本研究回答的问题 + 多组学策略概述
验证: 3段读完能回答: (1)为什么重要? (2)别人做了啥? (3)缺什么? (4)我们怎么补?
3.5 Discussion (Day 17-18)
目标: 5段，≤1500字，按7.4五段模型
Para 1: 核心发现总结
Para 2: 机制解读(间接调控)
Para 3: 旁分泌放大模型
Para 4: 疾病阶段特异性
Para 5: 局限性与展望
验证: Discussion中没有任何一句话是Results中没出现过的(除了限度和展望); 所有新claim都来自本研究数据
3.6 Abstract + Title (Day 19)
目标: Abstract ≤150 words, Title ≤150 chars
Title格式: [核心发现] in [疾病模型] reveals [机制]
Abstract: 1句背景 + 1句方法概述 + 3-4句核心结果 + 1句结论
验证: Abstract中每个数字都能在Results中找到出处

第三遍: 检查所有数字一致性(文中 vs Figure vs Supplement)

字数统计 (≤7000, 不含Methods/Refs/Legends)
Figure数量和分辨率 (≤7主图, 300dpi min)

Data Availability声明的accession全部可访问
验证: 逐项对照Cell Reports Author Guidelines打勾
4.3 Cover Letter
